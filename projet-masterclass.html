Automatisation de Masterclass

Le besoin : Industrialiser une application web

Le défi était de faire passer une application de son environnement de développement local à un serveur de production public. Cela impliquait de garantir sa disponibilité, sa performance et sa sécurité, simulant ainsi la mise en production d'une solution au sein d'un DataLab dans un contexte bancaire.

La solution technique : une infrastructure MLOps-Ready

J'ai mis en place une architecture robuste, prête à accueillir des applications data complexes comme des API de modèles de Machine Learning (développées en Python) :

  • Gestion de version (Git) : Utilisation de Git pour un versioning rigoureux du code, pilier de toute démarche CI/CD.
  • Déploiement sur serveur (VPS) : Configuration d'un environnement Linux pour un contrôle total de l'infrastructure.
  • Orchestration (Nginx & PM2) : Nginx sert de reverse proxy pour la sécurité et la performance, tandis que PM2 assure la haute disponibilité de l'application.
  • Conteneurisation (Docker) : Ce projet a été pensé pour être conteneurisé avec Docker. Cette étape est cruciale pour garantir la reproductibilité des environnements, un prérequis indispensable en DataOps/MLOps pour s'assurer qu'un modèle se comporte de la même manière en production qu'en développement.

Lien avec l'offre d'emploi

Cette expérience démontre ma maîtrise des outils et méthodologies fondamentales pour travailler au sein d'une équipe pluridisciplinaire (Data scientists, DevOps, etc.) :

  • ✔️ Git, CI/CD, DataOps/MLOps, docker : J'ai mis en pratique toute la chaîne DevOps nécessaire à l'industrialisation d'un projet, de la gestion du code à la mise en production.
  • ✔️ Environnement industriel bancaire : La rigueur appliquée (sécurité SSL, haute disponibilité) est directement transposable aux exigences d'une holding bancaire.
  • ✔️ Capacité d'abstraction : J'ai conçu une infrastructure capable non seulement d'héberger ce site, mais aussi d'accueillir de futures solutions d'IA générative.
